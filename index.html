How to use it
Как подключить

Three ways to pull a logo Три способа достать логотип

1 · Direct CDN URL 1 · Прямой URL CDN

The simplest form. Works in <img src>, CSS background-image, email templates, anywhere. Самое простое. Работает в <img src>, CSS background-image, email-шаблонах, где угодно. 

https://cdn.jsdelivr.net/gh/
  maslovsa/kyt-entity-registry@main/
  logos/<dir>/<slug>.png

2 · Fuzzy resolver (JS) 2 · Нечёткий резолвер (JS)

For freeform labels like "Binance Hot Wallet 10". One fetch per session, keyword-overlap scoring. Для свободных лейблов вроде "Binance Hot Wallet 10". Один фетч за сессию, scoring по пересечению ключевых слов. 

// lookup.js from the registry
import { createLookup }
  from '/lookup.js'

const l = await createLookup()
l.resolve({
  category: 'dex',
  label: 'Uniswap V3 Deposits',
})
// → { url, slug, name, … }

3 · MCP server (for Claude) 3 · MCP-сервер (для Claude)

Add one block to your ~/.claude/settings.json. Claude agents call resolve_logo directly. Одна запись в ~/.claude/settings.json и Claude-агент сам вызывает resolve_logo. 

"kyt-entity-registry": {
  "command": "uvx",
  "args": [
    "--from",
    "git+https://github.com/maslovsa/
      kyt-entity-registry.git
      #subdirectory=mcp/python",
    "kyt-entity-registry-mcp"
  ]
}
